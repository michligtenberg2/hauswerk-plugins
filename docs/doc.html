Introductie

Hauswerkis een open-source GUI-applicatie, gebouwd met Python (PyQt6), waarmee je op modulaire wijze video- en audiobewerkingen kunt uitvoeren via plugins. Zie het als een digitale gereedschapskist: de hoofdinterface bestaat uit verschillende tabbladen, elk met een specifieke tool (plugin). Standaard biedt Hauswerk diverse officiële plugins zoals Collage, Clipper, Stretcher en Concat – elk tabblad vertegenwoordigt een andere functionaliteit (bijv. een collage van video’s maken, clips random knippen, video’s vertragen, video’s aan elkaar plakken, enz.). Daarnaast is er een algemeen Dashboard-tabblad voor overzicht en mogelijk een Plugin Store-tabblad om nieuwe plugins te ontdekken en te installeren. Het kernidee van Hauswerk is flexibiliteit: nieuwe functies kunnen eenvoudig worden toegevoegd als plugin. Een plugin is in feite een Python-module die een Qt-widget ( QWidget) bevat met eigen GUI-logica. De Hauswerk-app laadt zulke plugins dynamisch en integreert de plugin-widgets naadloos in de interface (meestal als nieuwe tabbladen of menu-items). Officiële plugins (zoals de bovengenoemde tools) worden via de ingebouwde plugin-browser aangeboden, en gebruikers kunnen ook community-plugins installeren of zelf ontwikkelen. Onder de motorkap biedt Hauswerk enkele centrale hulpmiddelen aan plugins: Een SettingsManager voor applicatie-instellingen (zie settings.py) – hiermee kunnen plugins globale instellingen lezen/schrijven, opgeslagen in een JSON-configuratiebestand (standaard ~/.hauswerk_settings.json). Een Thema/Style manager (zie style.py) voor thema-beheer – zorgt voor consistente styling (licht/donker thema) van de GUI en plugins. Een Preset-systeem voor het opslaan/herladen van plugin-specifieke configuraties (bijv. de positie van clips in de Collage-plugin) via JSON-bestanden. Een ThumbnailWorker en FFmpegWorker mechanisme voor achtergrondprocessen – hiermee kunnen plugins zware taken (zoals videoverwerking via FFmpeg of het genereren van thumbnails) in aparte threads uitvoeren zodat de GUI responsief blijft:contentReference[oaicite:0]{index=0}:contentReference[oaicite:1]{index=1}. In deze documentatie leer je hoe je Hauswerk-plugins kunt ontwikkelen, uploaden en gebruiken. We bespreken de plugin-structuur, de integratie met de Hauswerk GUI, het gebruik van presets/thumbnails/workers, en we geven best practices voor validatie en foutafhandeling. Of je nu een gebruiker bent die een plugin wil installeren of een ontwikkelaar die er een wil bouwen – deze gids helpt je op weg.

📦 Wat bevat Supabase?

  • Één .py-bestand met een QWidget-subklasse
  • plugin.json met metadata (entry, class, tags, etc.)
  • Optioneel: .svg-icoon en preview.png
  • SHA256-checksum en automatische validatie
  • Downloadbaar .hauswerk-plugin.zip

🔍 ZIP Structuur

{
  "name": "Collage",
  "entry": "collage.py",
  "class": "CollageWidget",
  "tags": ["video", "overlay"],
  "verified": true,
  "slug": "collage",
  "icon": "collage.svg",
  "preview": "collage.png",
  "download_url": "https://.../collage.hauswerk-plugin.zip"
}

🚀 Quick Start

Volg deze stappen om snel een plugin te installeren:

  1. Open Hauswerk en ga naar de Plugin Store.
  2. Zoek naar Collage en klik op Installeren.
  3. Herstart de applicatie; de plugin verschijnt in je menu.

Voor ontwikkeling gebruik je:

hauswerk plugin new --name MijnPlugin

🛠️ API Referentie

upload_plugin_to_supabase.py

CLI-tool voor upload:

usage: upload_plugin_to_supabase.py --bucket BUCKET --file ZIPFILE

Parameters:

  • --bucket: naam van de Supabase-bucket
  • --file: pad naar de plugin ZIP

plugin_browser.js

Belangrijke functies:

  • fetchPlugins(): haalt lijst op via /rest/v1/plugins
  • renderPlugin(item): maakt DOM-elementen voor elke plugin

⚠️ Troubleshooting

Checksum mismatch: controleer of de ZIP niet beschadigd is.

Network errors: zorg voor internettoegang en correcte CORS-configuratie.
